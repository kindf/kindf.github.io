io复用--select

Jan 10, 2020
1 min read

select原理

  • 进程调用select时,fd_set会从用户空间拷贝到内核空间中去
  • 内核会为每个被监听的文件描述符注册回调函数
  • 遍历所监听的文件描述符,并做一次poll(驱动程序提供)操作(poll会将调用select的用户进程插入到对应设备驱动的等待队列中,并返回当前可用的资源)
  • 随后内核根据遍历后有无相应的资源可用来决定是否让进程睡眠
  • 当设备驱动发现自身资源变为可用后,就会将fd_set从内核拷贝到用户空间并唤醒该资源等待队列上的进程,select返回
  • 用户遍历fd_set后根据相应的改变进行操作

注意事项

  • select 函数调用前后会修改 readfds、writefds 和 exceptfds 这三个集合中的内容（如果有的话），所以如果您想下次调用 select 复用这个变量，记得在下次调用前再次调用 select 前先使用 FD_ZERO 将集合清零，然后调用 FD_SET 将需要检测事件的 fd 再次添加进去
  • select 函数也会修改 timeval 结构体的值，这也要求我们如果像复用这个变量，必须给 timeval 变量重新设置值
  • select 函数的 timeval 结构体的 tv_sec 和 tv_usec 如果两个值设置为 0，即检测事件总时间设置为0,则select会立即返回
  • 如果将 select 函数的 timeval 参数设置为 NULL，则 select 函数会一直阻塞下去，直到我们需要的事件触发
  • 在 Linux 平台上，select 函数的第一个参数必须设置成需要检测事件的所有 fd 中的最大值加1,在 Windows平台上，select函数的第一个值传任意值都可以

select相关API

FD_ZERO(fd_set *fdset):将fd_set变量的所有位初始化为0  
FD_SET(int fd, fd_set *fdset):在参数fdset指向的变量中注册文件描述符fd的信息  
FD_CLR(int fd, fd_set *fdset):从参数fdset指向的变量中清除文件描述符fd的信息  
FD_ISSET(int fd, fd_set *fdset):若参数fdset指向的变量中包含文件描述符fd的信息,返回true 

struct timeval
{
    long tv_sec; //seconds
    long tv_usec; //microsenconds
}

#include <sys/select.h>
#include <sys/time.h>
int select(int maxfd, fd_set *readset, fd_set *writeset, fd_set *exceptset, const struct timeval *timeout);
//maxfd 监视对象文件描述符数量
//readset 将所有关注"是否存在待读取数据"的文件描述注册到fd_set型变量,并传递其地址值
//writeset 将所有关注"是否可传输无阻塞数据"的文件描述注册到fd_set型变量,并传递其地址值
//exceptset 将所有关注"是否发生异常"的文件描述注册到fd_set型变量,并传递其地址值
//timeout 调用select函数后,为防止无限阻塞的状态,传递超时(time-out)信息
//返回值 发生错误时返回-1,超时返回时返回0。因发生关注的事件返回时,返回大于0的值,该值是发生事件的文件描述符数

select优缺点

缺点:

1.同时监听的文件描述符数量有限(maxfd的最大值为1024)
2.每次select返回都需要遍历所有监听的描述符
3.内存复制开销,select需要在用户空间与内核空间来回拷贝fd_set

优点:

1.程序具有兼容性(linux与windows都有类似的API)
